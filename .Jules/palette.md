## 2025-02-12 - Accessible Clear Buttons in Search Inputs
**Learning:** Custom search and filter inputs with `outline-none` (like in ProjectsApp, BlogApp, and Launcher) often break accessibility by removing the browser's native clear functionality. Furthermore, simply adding a clear button is insufficient if clicking it causes the user to lose keyboard focus context, forcing them to tab back through the DOM to type again.
**Action:** When adding custom clear buttons to text inputs, always implement them as `type="button"` (to prevent form submission), give them an explicit `aria-label` (e.g., "Clear filter"), ensure they have `focus-visible` utility classes for keyboard navigation, and critically, manage focus via a React `useRef` so that clicking the clear button immediately returns focus (`inputRef.current?.focus()`) to the input field.
