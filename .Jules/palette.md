## 2024-05-18 - Input Clear Buttons & Focus Management
**Learning:** Adding clear buttons to search or filter inputs can disrupt keyboard navigation if focus is not properly managed. When a user clicks a clear button or triggers it via keyboard, they expect to immediately start typing a new query. If focus is lost or remains on the clear button, the user experience is degraded.
**Action:** Always include an accessible 'clear' button for search or filter inputs, and ensure focus is explicitly returned to the associated input field (`inputRef.current?.focus()`) after clearing. Additionally, ensure the clear button itself has proper `focus-visible` styles (`focus-visible:outline focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[var(--color-amber)]`) to remain accessible to keyboard-only users.
