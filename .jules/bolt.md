## 2025-04-24 - Pre-compute Search Strings for React Rendering Loops
**Learning:** In a React application, frequently calling `.toLowerCase()` or concatenating strings during a render loop (such as inside a `useMemo` for filtering a list of items based on a search query) can degrade performance, especially as the list size grows. This is especially true for the `Launcher` component, which re-evaluates the `apps` list on every keystroke.
**Action:** Move the computation of the searchable string into the module load time or object creation phase (e.g., in `registry.ts` and `useFeaturedApps.ts`). When performing the search, lower-case the search query only once per query change instead of inside the `.filter()` loop, and use the pre-computed lower-cased target strings for comparison.
